# Delegation Protocol

The `dt` (Delegate Team) architecture is built on a strict **Claude-supervised MetaGPT delegation runtime**. This document defines the boundaries, responsibilities, and security policies of the nested agent orchestration.

## The 4-Layer Orchestration Model

1. **Controller (Supervisor)**: e.g., Claude Code
   - Responsible for understanding the user's task, writing the technical brief, determining constraints, and reviewing the final output.
   - **Authority**: Holds the final commit and merge authority.

2. **Policy Gateway**: `dt` (Delegate Team)
   - Decides whether a task should be handled by a single backend or routed to the multi-agent executor (MetaGPT).
   - Enforces execution limits: budgets, timeouts, maximum delegation depth, allowed tools, and workspace boundaries.

3. **Team Orchestrator**: MetaGPT
   - Takes the structured brief from the Gateway and decomposes it into distinct roles (e.g., Architect, Engineer, Reviewer, Tester).
   - Coordinates the assembly-line workflow among multiple worker agents, compiling partial outputs into a cohesive whole.

4. **Model Adapters (Workers)**: e.g., VertexCoder, Codex, MiniMax
   - Execute specific roles using the optimal model backend based on capability tags (not hardcoded names).

## Security & Execution Policies

To prevent excessive agency and ensure the supervisor remains in control, the orchestration runtime enforces the following policies:

### 1. Untrusted Worker Output & Prompt Injection Mitigation
All output generated by the MetaGPT workers is treated as **untrusted data**. Worker agents cannot inject instructions back to the Supervisor.

- The Gateway (`dt`) normalizes all results into a strict contract.
- The Supervisor (Claude Code) must receive results as data payload, not as executable instructions.
- Example payload restriction:
  ```json
  {
    "type": "worker_output",
    "trusted": false,
    "content": "...",
    "allowed_to_instruct_controller": false
  }
  ```

### 2. Authority Constraints
Workers and the Team Orchestrator execute the work, but they **cannot** finalize the task.
- MetaGPT executes.
- `dt` logs and orchestrates.
- Claude Code reviews the diff.
- Final commit/push requires Claude Code (or the Human) to pass the review gate.

### 3. Excessive Agency Prevention
MetaGPT and its workers operate under a highly restricted environment policy:
```json
{
  "filesystem": "workspace_only",
  "network": "deny_by_default",
  "dependency_install": "approval_required",
  "shell": "allowlist_only",
  "commit": "forbidden",
  "secrets_read": "forbidden"
}
```

By strictly separating orchestration from execution, the `dt` suite ensures that complex, multi-agent tasks are completed efficiently without compromising the security, stability, or final authority of the human-in-the-loop and the primary supervisor agent.
